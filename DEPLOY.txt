+ point GoDaddy DNS at it.


RECOMMENDED — CLOUDFLARE PAGES (free SSL, fast, form-friendly)
--------------------------------------------------------------------------------

STEP 1 — Deploy the site

  Option A — Dashboard upload
  1. Sign in at https://dash.cloudflare.com (free account) — enable 2FA
  2. Workers & Pages → Create → Pages → Upload assets
  3. Project name: ninjajoartworks
  4. Drag the entire website/ folder contents (index.html, css/, js/, assets/, etc.)
  5. Deploy → you get a preview URL like ninjajoartworks.pages.dev

  Option B — Wrangler CLI (from website/)
  1. Create Cloudflare API token (Pages deploy permission)
  2. $env:CLOUDFLARE_API_TOKEN = "token"  (PowerShell — session only)
  3. npm run deploy
  See GO-LIVE-NOW.txt for full gate checklist.

  Rebuild deploy zip (forward-slash paths — required for CSS on Cloudflare):
    python -c "import os,zipfile; r=r'D:\\ninja jo artworks\\website'; o=r'D:\\ninja jo artworks\\ninjajoartworks-site.zip'; ..."
    Or run from website folder after Grok rebuilds ninjajoartworks-site.zip at repo root.

  PLAIN TEXT / NO STYLING after deploy?
    → CSS path broken by Windows backslash zip. Redeploy fixed zip from repo root.

STEP 2 — Connect ninjajoartworks.com
  Option A — Use Cloudflare nameservers (simplest)
    1. Cloudflare Pages → your project → Custom domains → Add ninjajoartworks.com
    2. Cloudflare shows two nameservers (e.g. ada.ns.cloudflare.com)
    3. GoDaddy → My Products → ninjajoartworks.com → DNS → Nameservers
    4. Change to Custom → paste Cloudflare nameservers → Save
    5. Wait 15 min–48 hrs for propagation → https://ninjajoartworks.com works

  Option B — Keep GoDaddy DNS (advanced)
    1. In Cloudflare Pages custom domain setup, choose CNAME route
    2. GoDaddy → DNS → Add CNAME: www → your-project.pages.dev
    3. GoDaddy → DNS → Forward apex ninjajoartworks.com → https://www.ninjajoartworks.com
    4. Enable SSL in Cloudflare (automatic)

STEP 3 — www redirect
  Cloudflare Pages custom domain setup usually adds both apex + www.
  Verify https://www.ninjajoartworks.com redirects to apex (or vice versa).

FORM → ninjajoartworks@gmail.com (Web3Forms — 5 minutes)
--------------------------------------------------------------------------------
The form code is wired. One key activates live email delivery.

  1. Go to https://web3forms.com
  2. Sign up with ninjajoartworks@gmail.com
  3. Create an access key (free tier: 250 submissions/month)
  4. Open website/js/form-config.js
  5. Paste key into web3formsAccessKey: "YOUR_KEY_HERE"
  6. Re-deploy / re-upload website folder to Cloudflare Pages (or refresh hosting)

Test: submit a work request → confirmation page should say email was sent →
check ninjajoartworks@gmail.com inbox (and spam).

Until the key is set: form still validates and shows confirmation preview locally,
but no email is sent (banner on request page notes this).

TROUBLESHOOTING
--------------------------------------------------------------------------------
  Domain shows GoDaddy parking page
    → DNS not pointed at host yet; wait for propagation or fix nameservers

  SSL certificate error
    → On Cloudflare/Netlify, wait 15 min after adding custom domain

  Form submits but no email
    → Check form-config.js key · Web3Forms dashboard · spam folder

  Lane dropdown looks wrong after deploy
    → Hard refresh (Ctrl+F5) — browser may cache old request-form.js
